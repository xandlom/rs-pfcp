# v0.2.x Stabilization Roadmap

**Created:** 2025-12-03
**Status:** Active
**Target:** Stabilize v0.2.x with non-breaking improvements
**Strategy:** Let v0.2.0 mature, gather user feedback, add polish

---

## Philosophy

After the v0.2.0 breaking changes release, we're focusing on:
- **Stability**: Let the new API mature with real-world usage
- **Non-breaking**: All improvements are backward compatible
- **User feedback**: Respond to issues and feature requests
- **Polish**: Improve ergonomics and documentation

**Next breaking changes will be bundled in v0.3.0 (later)**

---

## Planned v0.2.x Releases

### v0.2.1 (Target: 2-3 weeks)
**Focus:** Quick wins and ergonomics

#### From API Improvements Index:

1. **Expand IntoIe Trait** (#6 from index)
   - Add `IntoIe` implementations for more types
   - Enable direct conversion: `builder.field(value)` instead of `builder.field(value.to_ie())`
   - Non-breaking: Adds new trait impls
   - Effort: 1 day
   - **Priority:** High (immediate ergonomic win)

2. **Default Trait Implementations** (#7 from index)
   - Add `Default` for all builders
   - Enable: `let builder = MyBuilder::default().field(x).build()?`
   - Non-breaking: Adds trait impls
   - Effort: 0.5 day
   - **Priority:** Medium (nice Rust idiom)

3. **Builder Documentation** (#9 from index)
   - Comprehensive builder guide in docs/guides/
   - Document all builder patterns
   - Show common use cases
   - Non-breaking: Pure documentation
   - Effort: 1 day
   - **Priority:** High (helps users)

#### Additional Quality Improvements:

4. **Example Enhancements**
   - Add more real-world examples
   - Improve existing example documentation
   - Add troubleshooting section
   - Non-breaking: Pure examples
   - Effort: 0.5 day
   - **Priority:** Medium

**Total Effort:** ~3 days
**Breaking:** No

## Deferred to v0.3.0 (Breaking Changes)

These items require breaking changes and will be bundled:

1. **Custom Error Type (PfcpError)** - High priority but breaking
   - Replace `io::Error` with structured errors
   - Better debugging and recovery
   - Effort: 3-4 days

2. **Newtype Wrappers** - Medium priority, breaking
   - Wrap primitive types to prevent argument swapping
   - Type safety improvements
   - Effort: 1-2 days

**Strategy:** Wait for user feedback on v0.2.x, then bundle all breaking changes
